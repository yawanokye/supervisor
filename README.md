# ProjectReady AI Supervisor Assistant MVP 0.4

This build reviews initial thesis chapters, revised chapters, research proposals, and complete theses against the official 109-item thesis self-evaluation checklist.

## New in version 0.4

### Revised chapter review

Students can choose either:

- **Initial chapter**
- **Revised chapter**

When **Revised chapter** is selected, the student must provide the supervisor comments by:

- uploading one or more DOCX or PDF files,
- pasting the comments into a text box, or
- using both methods.

The system can read:

- ordinary supervisor-comment documents,
- comments in tables,
- green square-bracketed comments generated by this app,
- labelled comments such as `Supervisor comment:` or `Action required:`, and
- native Microsoft Word comments stored in `comments.xml`.

The student may also upload the original chapter. This is optional but recommended because the system can then compare the most relevant original and revised passages.

### Revision decisions

Each extracted supervisor comment receives one of four decisions:

- **Addressed**
- **Partly addressed**
- **Not addressed**
- **Manual confirmation required**

The dashboard and Word report show:

- the original supervisor comment,
- the comment source,
- best matching evidence in the revised chapter,
- original-to-revised comparison where available,
- confidence level,
- required follow-up action, and
- an overall supervisor-comment compliance score.

The revised DOCX output also marks unresolved passages in red and inserts a green square-bracketed follow-up comment immediately after the affected text.

## Existing workflow

- Chapter One can be reviewed as Chapter One only or as a research proposal.
- Chapters Two to Five require earlier chapters for alignment.
- Earlier chapters may be uploaded as one composite file or several separate files.
- Only the current chapter is annotated.
- Previous chapters, supervisor comments, and the optional original chapter are read for comparison but are not modified.

## Scoring

For an initial chapter with earlier chapters:

- 80% official checklist
- 20% cross-chapter alignment

For a revised Chapter One or revised proposal without alignment:

- 80% official checklist
- 20% supervisor-comment compliance

For revised Chapters Two to Five:

- 65% official checklist
- 15% cross-chapter alignment
- 20% supervisor-comment compliance

A revised chapter cannot be marked ready while a supervisor comment remains **Not addressed**.

## Run locally

```bash
python -m venv .venv
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Open `http://127.0.0.1:8000`.

## Render deployment

```bash
pip install -r requirements.txt
```

```bash
uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

Health-check path:

```text
/health
```

## MVP limitations

1. The supervisor-comment compliance engine is evidence-based and rule-driven. Complex conceptual comments may still require manual confirmation.
2. DOCX files provide stable paragraph references but not stable page numbers. PDF provides stable pages.
3. Scanned PDFs are not supported.
4. The in-memory cache must be replaced with Redis or a database before multi-instance commercial deployment.
5. Native Word comment anchors are not yet mapped to their exact original sentence. The comment text itself is extracted and assessed.
